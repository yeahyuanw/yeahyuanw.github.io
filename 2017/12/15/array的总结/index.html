    • 380.Insert Delete GetRandom O(1)
    • 381.Insert Delete GetRandom O(1) - Duplicates allowed
      这两个问题是需要设计一个数据结构, 对于插入、删除，和随机读取的时间复杂度都为1.
      我们知道HashMap的插入和删除的时间复杂度都为O(1)，但是不能随机读取; 数组随机读取的时间复杂度是O(1)，但是插入和删除的时间复杂度是O(n).这两个问题就需要将HashMap和数组结合起来设计一个新的数据结构.
      对于381刚开始我理解错了题意，以为remove(val)是要删除所有的val, 后来通过阅读Discuss才发现只需要删除一个val.
      要将HashMap和ArrayList结合起来, 需要解决ArrayList的插入和删除时间复杂度为O(n)的问题, 我们知道如果只在ArrayList的末尾进行插入和删除，那么时间复杂度则是O(1).所以我们需要利用这一特性.每次将需要删除的元素与ArrayList末尾的元素进行替换, 并且更新map， 然后分别删除map中的val和ArrayList的末尾.
      对于第381题, 可以定义Map>, List; map中的key是val, value是一个列表，保存了val在list中的索引.而list中保存的是pair
      思考问题: 如果remove(val)是要删除所有的val，该如何设计？是否还能做到O(1)?

    • 719.Find K-th Smallest Pair Distance
      这个题目先进行排序, 则可知largest pair distance 为nums[nums.length - 1] - nums[0], 然后再遍历nums求得smallest pair distance, 然后再运用二分搜索, mid_value = (largest + smallest)/2， 然 后统计小于mid_value的pair distance有多少个, 跟k值进行比较从而缩小二分搜索的范围.
  • dp
    • 718.Maximum Length of Repeated Subarray
      这个题目与Longest Common Substring的状态转移函数一样:
      dp[i][j] = dp[i-1][j-1] + 1 if A[i] == B[i] and dp[i][j] = 0 if A[i] != B[i]
      然后取最大的dp[i][j] : maximum = Math.max(maximum, dp[i][j]);
